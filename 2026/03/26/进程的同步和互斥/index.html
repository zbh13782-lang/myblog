进程的同步和互斥

同步

进程之间有严格的执行顺序比如
读进程必须等写进程完成

互斥

多个进程想要抢占同一个临界资源（打印机，摄像头等io设备)
一个进程历经进入区，临界区，结束区，剩余区
(上锁) (使用) (解锁) （其他剩余任务）

进程互斥的解决办法

软件方法

单标志法

开始：int turn = 0

进程0：
while(turn!=0);// 不是0就一直等着
do sth//临界区
turn = 1//结束
do sth//剩余任务

进程1：
while(turn!=1)
…..

问题：进程0访问之后把标志设置成1，要是1不在的话，又执行进程0就会阻塞

双标志

bool flag[2]
进程0：
while(!flag[1]);
flag[0]=true
do sth
flag[0]=false
do sth

问题：当两个进程并发执行，仍然会出现同时访问情况

Peterson算法

单标志+双标志
flag[2]
turn

p0: flag[0] = true
turn = 1
while(flag[0] && turn= =0)

do sth…

硬件方法

中断

把请求操作设置成原子性的，就是说当前cpu只能跑这个进程，不能并发别的

问题：如果有多个cpu，其他cpu上的进程也要访问临界资源就出现问题了

TS

test and set
把临界区加锁，等一个进程结束了解锁
tsl指令：

当lock本来是false，没有占有时候，while没卡住，同时把lock锁了，这样其他进程访问的时候执行while卡住了，等临界资源释放出来在用

问题：一直等待
